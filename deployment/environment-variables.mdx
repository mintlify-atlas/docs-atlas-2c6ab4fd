---
title: 'Environment variables'
description: 'Reference for all environment variables used by Condo.'
---

Condo is configured entirely through environment variables. In local development, these are managed in a `.env` file generated by `bin/prepare.js`. In production, set them directly in your environment, Docker Compose file, or Dokku config.

<Note>
Use `.env.example` at the repo root as the canonical template. Never commit a `.env` file containing real secrets.
</Note>

## Database

<ParamField path="DATABASE_URL" type="string" required>
  PostgreSQL connection string. Set automatically by Dokku's `postgres:link` command, or configure manually.

  **Example:** `postgresql://postgres:postgres@localhost/main`
</ParamField>

<ParamField path="REDIS_URL" type="string">
  Redis connection URL used for sessions, async task queues (Bull), and caching.

  **Example:** `redis://localhost:6379`
</ParamField>

<ParamField path="KV_URL" type="string">
  Alternative to `REDIS_URL` for Valkey or Redis-compatible key-value stores.

  **Example:** `redis://localhost:7001`
</ParamField>

## Server

<ParamField path="SERVER_URL" type="string" required>
  The public-facing URL of the application. Used by Next.js for API calls from the frontend to the backend.

  **Example:** `https://condo.example.com`
</ParamField>

<ParamField path="PORT" type="number">
  The port the application listens on. Assigned automatically by `bin/prepare.js` during local setup.

  **Example:** `4006`
</ParamField>

<ParamField path="NODE_ENV" type="string" default="development">
  Application environment. Must be set to `production` in production deployments.

  **Accepted values:** `development`, `production`, `test`
</ParamField>

<ParamField path="DEFAULT_LOCALE" type="string" default="en">
  Default locale for the application UI.

  **Example:** `en`
</ParamField>

<ParamField path="DISABLE_LOGGING" type="boolean" default="false">
  Disables structured logging output when set to `true`. Not recommended in production.

  **Example:** `false`
</ParamField>

<ParamField path="DISABLE_RATE_LIMIT" type="boolean" default="true">
  Disables API rate limiting. Set to `false` in production to enable rate limiting.

  **Example:** `true`
</ParamField>

## Auth and security

<ParamField path="COOKIE_SECRET" type="string" required>
  Secret key used by KeystoneJS to sign session cookies. Use a long, randomly generated string.

  **Example:** `AWJfbsbaf!` (use a securely generated 32+ character string in production)
</ParamField>

<ParamField path="DATA_ENCRYPTION_CONFIG" type="string">
  JSON configuration for data encryption used by the webhook system. The object key is the version identifier and must match `DATA_ENCRYPTION_VERSION_ID`. The `secret` must be exactly 32 characters.

  **Example:** `{"myapp_1":{"algorithm":"aes-256-gcm","secret":"your-32-character-secret-string","compressor":"brotli","keyDeriver":"pbkdf2-sha512"}}`
</ParamField>

<ParamField path="DATA_ENCRYPTION_VERSION_ID" type="string">
  Identifies which encryption key version from `DATA_ENCRYPTION_CONFIG` is currently active.

  **Example:** `v1`
</ParamField>

## Build variables

These variables are used during the Docker image build stage. Set them in your `.env` before running `docker-compose build`.

<ParamField path="DOCKER_FILE_INSTALL_COMMAND" type="string">
  Shell command run inside the Dockerfile to install extra system packages or Python dependencies.

  **Example:** `python3 -m pip install 'psycopg2-binary>=2.8.5' && python3 -m pip install 'Django>=3.0.6'`
</ParamField>

<ParamField path="DOCKER_FILE_BUILD_COMMAND" type="string">
  Shell command run inside the Dockerfile to build static assets or prepare production files.

  **Example:** `yarn workspace @app/condo build`
</ParamField>

## Runtime (Docker Compose)

These variables are consumed by `docker-compose.yml` at container startup.

<ParamField path="DOCKER_COMPOSE_START_APP_COMMAND" type="string" required>
  The command Docker uses to start the web application inside the container.

  **Example:** `yarn workspace @app/condo start`
</ParamField>

<ParamField path="DOCKER_COMPOSE_COOKIE_SECRET" type="string" required>
  Session cookie secret passed into the container. Equivalent to `COOKIE_SECRET` for Docker Compose deployments.

  **Example:** `some-secret-random-string`
</ParamField>

<ParamField path="DOCKER_COMPOSE_SERVER_URL" type="string" required>
  Public URL of the application passed into the container. Equivalent to `SERVER_URL` for Docker Compose deployments.

  **Example:** `https://condo.example.com`
</ParamField>

## File storage

<ParamField path="FILE_FIELD_ADAPTER" type="string" default="local">
  Storage backend for uploaded files. Use `local` for development or `s3` for production.

  **Accepted values:** `local`, `s3`
</ParamField>

<ParamField path="S3_ENDPOINT" type="string">
  S3-compatible endpoint URL. Required when `FILE_FIELD_ADAPTER=s3`.

  **Example:** `https://s3.amazonaws.com`
</ParamField>

<ParamField path="S3_ACCESS_KEY_ID" type="string">
  AWS or S3-compatible access key ID. Required when `FILE_FIELD_ADAPTER=s3`.
</ParamField>

<ParamField path="S3_SECRET_ACCESS_KEY" type="string">
  AWS or S3-compatible secret access key. Required when `FILE_FIELD_ADAPTER=s3`.
</ParamField>

<ParamField path="S3_BUCKET" type="string">
  Name of the S3 bucket for file storage. Required when `FILE_FIELD_ADAPTER=s3`.
</ParamField>

<ParamField path="S3_REGION" type="string">
  AWS region for the S3 bucket.

  **Example:** `us-east-1`
</ParamField>

## Worker

<ParamField path="WORKER_CONCURRENCY" type="number" default="50">
  Number of concurrent jobs the worker process will handle.

  **Example:** `50`
</ParamField>

## Cache

<ParamField path="ADAPTER_CACHE_CONFIG" type="string">
  JSON configuration for the Keystone adapter-level cache.

  **Example:** `{"enabled": true, "excludedLists":[], "logging":0, "maxCacheSize":1000, "logStatsEachSecs": 60}`
</ParamField>

<ParamField path="REQUEST_CACHE_CONFIG" type="string">
  JSON configuration for per-request caching.

  **Example:** `{"enabled": true, "logging":0, "logStatsEachSecs": 60}`
</ParamField>

## Notifications

<ParamField path="NOTIFICATION__SEND_ALL_MESSAGES_TO_CONSOLE" type="boolean" default="true">
  When `true`, all notification messages are printed to stdout instead of being dispatched. Useful for local development.

  **Example:** `true`
</ParamField>

<ParamField path="NOTIFICATION__DISABLE_LOGGING" type="boolean" default="false">
  Disables notification-specific log output.

  **Example:** `false`
</ParamField>

## Optional integrations

<ParamField path="FIREBASE_CONFIG_JSON" type="string">
  Firebase Admin SDK service account JSON for push notifications. Retrieve from the Firebase console under **Settings → Service accounts**.

  **Example:** `{"type": "service_account", "project_id": "<PROJECT_ID>", ...}`
</ParamField>

<ParamField path="HMS_CONFIG_JSON" type="string">
  Huawei Messaging System (HMS) credentials for push notifications to Huawei devices.

  **Example:** `{"resident": {"clientId": "<ID>", "secret": "<SECRET>"}, "master": {"clientId": "<ID>", "secret": "<SECRET>"}}`
</ParamField>

<ParamField path="APPLE_CONFIG_JSON" type="string">
  Apple Push Notification service (APNs) credentials.

  **Example:** `{"kid": "<KEY_ID>", "iss": "<ISSUER_ID>", "privateKey": "<PRIVATE_KEY>"}`
</ParamField>

<ParamField path="HCAPTCHA_CONFIG" type="string">
  hCaptcha configuration for bot protection on forms.

  **Example:** `{"API_KEY":"<API_KEY>", "SITE_KEY":"<SITE_KEY>", "ANDROID_KEY":"<KEY>", "IOS_KEY":"<KEY>"}`
</ParamField>

<ParamField path="DISABLE_CAPTCHA" type="boolean" default="true">
  Disables captcha verification. Set to `false` in production when `HCAPTCHA_CONFIG` is configured.

  **Example:** `true`
</ParamField>

<ParamField path="OIDC_CONDO_CLIENT_CONFIG" type="string">
  OpenID Connect client configuration for SSO integration.

  **Example:** `{"serverUrl":"http://localhost:3000", "clientId":"<client id>", "clientSecret":"<client secret>"}`
</ParamField>

<ParamField path="FEATURE_TOGGLE_CONFIG" type="string">
  Feature flag service configuration. Supports remote API or static flags.

  **Example:** `{"url": "http://localhost:3100/api/features", "apiKey": "key"}`
</ParamField>

<ParamField path="MESSAGING_CONFIG" type="string">
  NATS-based messaging broker configuration for real-time subscriptions.

  **Example:** `{"enabled":true,"brokerUrl":"nats://127.0.0.1:4222","wsUrl":"ws://localhost:8080","tokenSecret":"your-secret-key",...}`
</ParamField>

<ParamField path="AI_ENABLED" type="boolean" default="true">
  Enables AI-powered features.

  **Example:** `true`
</ParamField>

<ParamField path="AI_FLOWS_CONFIG" type="string">
  JSON configuration mapping AI flow names to adapter endpoints (Flowise, n8n, etc.).
</ParamField>

<ParamField path="AI_ADAPTERS_CONFIG" type="string">
  JSON configuration for AI adapter credentials.
</ParamField>

<ParamField path="HELP_REQUISITES" type="string">
  Contact details shown in the UI for support requests.

  **Example:** `{"support_email": "help@example.com", "support_phone": "+1 301 000-00-00"}`
</ParamField>

<ParamField path="BANK_ACCOUNT_REQUEST_EMAIL_TARGET" type="string">
  Email address that receives bank account request notifications.

  **Example:** `help@example.com`
</ParamField>

<ParamField path="NEXT_TELEMETRY_DISABLED" type="number" default="1">
  Disables Next.js anonymous telemetry. Set to `1` to opt out.

  **Example:** `1`
</ParamField>

<ParamField path="ENABLE_SUBSCRIPTIONS" type="boolean" default="true">
  Enables GraphQL subscription support via the messaging broker.

  **Example:** `true`
</ParamField>

<ParamField path="USE_LOCAL_FEATURE_FLAGS" type="boolean" default="true">
  Forces the app to use locally defined feature flags instead of a remote service.

  **Example:** `true`
</ParamField>
